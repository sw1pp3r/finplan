# finplan

[English](README.md) · **Русский**

Самостоятельно размещаемый **планировщик cash-flow** — это не трекер расходов. finplan
отвечает на один вопрос: *хватит ли баланса, чтобы не пробить подушку на горизонте
планирования, а если нет — где провал и насколько глубокий?*

Вы вносите, где находитесь сегодня (**снимок**), что предстоит заплатить
(**обязательства**) и что ожидаете получить (**поступления**). finplan проецирует баланс
вперёд и показывает кривую, три сценария и любой разрыв.

- **Деплой через compose:** бэкенд FastAPI + SPA на React/Tailwind в одном образе + PostgreSQL.
- **Паритет с продом:** по умолчанию встроенный **PostgreSQL** — тот же движок, на котором
  приложение тестируется и работает в проде, так что сюрпризов «на SQLite прошло, на
  Postgres упало» нет. SQLite остаётся доступен для минимальной однофайловой установки.
- **Приватность by design:** ни телеметрии, ни аналитики. Единственный исходящий запрос —
  необязательное обновление курсов валют раз в сутки, и оно **выключено по умолчанию**.
- **Мультивалютность:** базовая валюта и курсы (ежедневные или вручную).

> ([README.md](README.md)) — переводы приветствуются.

## Быстрый старт (одна команда)

Нужен Docker с плагином Compose.

```bash
git clone https://github.com/sw1pp3r/finplan.git
cd finplan
docker compose up -d --build
```

Запустятся **PostgreSQL и приложение**. Откройте **http://localhost:8742**. При первом
запуске встроенный онбординг проведёт вас по модели и шагам настройки. Данные лежат в
Docker-томах (`finplan-db`, `finplan-images`) и переживают перезапуски и редеплои.

Это всё — для приватного/локального запуска `.env` не нужен. Чтобы что-то поменять (свои
креды БД, внешняя база или SQLite, обновление курсов, токен API), скопируйте
`.env.example` в `.env`, отредактируйте и снова выполните `docker compose up -d`.

## Откуда берутся числа

finplan — это движок прогноза, а не журнал операций. Нет импорта банковских выписок и
категоризации транзакций — вы ведёте несколько небольших осмысленных входных данных, а
проекцию строит приложение.

- **Снимок → точка отсчёта (T0).** На вкладке *Снимок* вы записываете текущий остаток по
  каждому счёту. Самый свежий снимок — это **T0**, откуда стартует прогноз. Несколько
  снимков со временем дают **скорость прожигания** (burn) — насколько в неделю тает баланс.
- **Поступления → ожидаемые доходы.** На вкладке *Доходы* перечислены будущие приходы
  (инвойсы, зарплата, оплаты). Пока статус *ожидается* — они поднимают кривую прогноза в
  свою дату. Когда деньги пришли, переводите строку в *получено*, и она уже отражена в
  следующем снимке.
- **Обязательства → будущие списания.** На вкладке *Расходы* — аренда, налоги, подписки,
  разовые платежи. Разовые вычитаются в свою дату; повторяющиеся (еженедельно / ежемесячно
  / ежегодно) — на каждую дату до конца горизонта.
- **Прогноз и разрыв (gap).** *Дашборд* строит баланс вперёд:
  `T0 − burn − обязательства + ожидаемые доходы`. Если кривая опускается ниже вашей
  **подушки** (cushion), finplan показывает **разрыв** — сколько не хватает и к какой дате.
  Три сценария (пессимистичный / базовый / оптимистичный) взвешивают доходы по вероятности.

Ещё есть песочница для экономики курса/продукта и визуальная **доска желаний** — ни то, ни
другое не влияет на прогноз, пока вы не переведёте желание в обязательство.

Встроенный **демо-режим** (тумблер в шапке) показывает приложение на правдоподобных
тестовых данных в отдельной in-memory базе; ваши настоящие данные при этом не трогаются.

## Конфигурация

Всё опционально — см. `.env.example`. Главное:

| Переменная | По умолчанию | Что делает |
|---|---|---|
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` | `finplan` | креды встроенного PostgreSQL; из них собирается `DATABASE_URL` приложения |
| `DATABASE_URL` | встроенный Postgres | переопределите, чтобы указать на внешнюю базу или SQLite |
| `FINPLAN_FX_AUTOFETCH` | `0` | `1` включает ежедневное обновление курсов с open.er-api.com. `0` = никакого исходящего трафика |
| `FINPLAN_API_TOKEN` | пусто | если задан, `/api/*` требует `Authorization: Bearer <token>` |
| `FINPLAN_IMAGE_DIR` | `/srv/finplan/wish-images` | где хранятся картинки доски желаний |

Схема создаётся автоматически при старте (лёгкие миграции, Alembic не нужен).
**Внешняя база:** задайте `DATABASE_URL` на свой Postgres, например
`postgresql+psycopg2://USER:SECRET@your-host:5432/finplan`. **Минимальный SQLite:** задайте
`DATABASE_URL=sqlite:////data/finplan.db` и смонтируйте том на `/data` — удобно для быстрой
однофайловой пробы, но для паритета с продом рекомендуется PostgreSQL.

## Обновление

Workflow релиза публикует Docker-образ в GHCR на каждый push в `main`. Чтобы обновить
запущенный деплой до последнего опубликованного образа:

```bash
docker compose pull
docker compose up -d
```

Ваши тома (данные и картинки) сохраняются при обновлении.

Чтобы зафиксировать конкретную сборку вместо `latest`, поставьте в `docker-compose.yml`
тег образа = SHA коммита или тег релиза, опубликованный workflow.

## Разработка (без Docker)

Бэкенд (Python 3.12+):

```bash
python -m venv .venv && . .venv/bin/activate
pip install fastapi "uvicorn[standard]" jinja2 sqlalchemy httpx python-multipart apscheduler pillow pytest
pytest -q                                   # прогнать тесты
DATABASE_URL=sqlite:///./dev.db FINPLAN_FX_AUTOFETCH=0 uvicorn app.asgi:app --port 8741
```

Фронтенд (Node 20+):

```bash
cd web
npm install
npm run build        # собирает web/dist, который раздаёт бэкенд
# или: npm run dev    # dev-сервер Vite с hot reload
```

Бэкенд раздаёт собранную SPA из `web/dist`, поэтому сделайте `npm run build` и запустите
uvicorn — либо держите dev-сервер Vite рядом.

## Приватность

finplan создан, чтобы работать на вашей машине или приватном сервере и оставлять ваши
финансовые данные вашими. Ни телеметрии, ни аналитики. Единственный необязательный
исходящий запрос — ежедневное обновление курсов, по умолчанию выключенное
(`FINPLAN_FX_AUTOFETCH=0`). С ним выключенным приложение вообще не ходит в сеть.
